# RWAI Arena - 视觉设计规范

> 定义网站的整体视觉风格、设计理念和视觉特征

---

**信任可靠**
- 强调验证标识

**结果导向**

```
Minimalist  Professional  Trustworthy  Data-Driven  Modern
```

## 2. 视觉特征

### 2.1 整体风格

**充足留白**
- 内容呼吸感强
- 避免拥挤的布局
- 清晰的信息层级

**清晰层级**
- 视觉重量明确
- 信息优先级清晰
- 引导用户视线

**数据驱动**
- 图表和指标突出
- 可视化信息呈现
- 量化成果展示

**专业配色**
- 主色调统一
- 语义色明确
- 品牌识别度高

### 2.2 视觉语言

**线条风格**
- 轮廓线图标（outline icons）
- 细线边框（1-2px）
- 干净的分割线

**形状使用**
- 圆角统一（6-16px）
- 卡片化设计
- Pill形状标签

**质感处理**
- 扁平化为主
- 轻微阴影增加层次
- 渐变色谨慎使用

## 3. 品牌元素

### 3.1 Logo规范

**RWAI Arena Logo**
- 位置：左上角（导航栏）
- 颜色：主色（#155EEF）或深色（#0F172A）
- 字重：Bold (700)
- 字号：20-24px
- 间距：右侧与其他导航元素保持16-24px

### 3.2 品牌色

**主色（Primary）: #155EEF**
- 用途：Logo、主按钮、链接、强调元素
- 象征：信任、专业、科技

**辅助色（Accent）: #0F4761**
- 用途：标题、副标题、重要文本
- 象征：稳重、深度

### 3.3 标识系统

**验证标识（Verified Badge）**
- 图标：✓
- 颜色：成功绿（#10B981）
- 背景：浅绿（#DCFCE7）
- 形状：Pill badge
- 意义：经过验证的最佳实践

**开源标识（Open Source Badge）**
- 图标：GitHub logo 或代码图标
- 颜色：深色
- 位置：Hero区、页脚
- 意义：完全开源、可复制

## 4. 情感化设计

### 4.1 交互反馈

**即时响应**
- Hover状态：颜色变化、阴影增强
- Active状态：轻微缩放
- Loading状态：加载动画

**过渡动画**
- 时间：200-300ms（默认）
- 缓动：ease-out
- 自然流畅

### 4.2 微交互

**按钮交互**
- Hover：背景色加深、轻微上浮
- Click：轻微缩放（scale: 0.98）
- Focus：外发光或边框高亮

**卡片交互**
- Hover：上浮4px、阴影增强
- 过渡：all 0.2s ease-out

**链接交互**
- Hover：颜色变深、可选下划线
- Focus：outline指示器

## 5. 视觉层次

### 5.1 字重层级

```
Bold (700)    - 页面主标题、重要强调
SemiBold (600) - Section标题、卡片标题、按钮
Medium (500)   - 导航链接、小标题、标签
Regular (400)  - 正文、描述
```

### 5.2 字号层级

```
Hero标题:   48-64px   (桌面) / 32-40px   (移动)
Section标题: 36-48px   (桌面) / 28-36px   (移动)
卡片标题:   24-32px   (桌面) / 20-24px   (移动)
小标题:     18-20px   (桌面) / 16-18px   (移动)
正文:       16px
次要文本:   14px
说明文本:   12px
```

### 5.3 颜色层级

```
主文本色: #0F172A  - 标题、重要正文
次文本色: #64748B  - 描述、说明
主色:     #155EEF  - 强调、链接、按钮
成功色:   #10B981  - 验证标识、成功状态
```

## 6. 空间设计

### 6.1 间距系统

```
Section间距:    80-120px  (页面各section之间)
卡片间距:      24-32px   (网格卡片)
元素间距:      16-24px   (卡片内元素)
小元素间距:    8-12px    (相关联的小元素)
```

### 6.2 容器宽度

```
全宽:          100%      (背景色块、Hero背景)
主容器:        1400px    (主要内容容器)
文本容器:      1200px    (文本内容区)
窄容器:        800px     (表单、详情)
移动端:        100%      (最大600px)
```

### 6.3 内边距

```
页面内容:      32px      (桌面) / 16px  (移动)
卡片内边距:    24px      (桌面) / 16px  (移动)
按钮内边距:    12-24px   (水平)
输入框内边距:  0 16px    (左右)
```

## 7. 视觉一致性

### 7.1 重复元素

**圆角统一**
- 按钮：6-8px
- 卡片：12-16px
- 徽章：12-16px（pill形状）
- 输入框：6-8px

**阴影统一**
- 卡片默认：无或极浅
- 卡片Hover：中等阴影
- 焦点状态：外发光

**边框统一**
- 分割线：1px solid #E2E8F0
- 输入框：1px solid #E2E8F0
- 次级按钮：2px solid #E2E8F0

### 7.2 布局模式

**网格系统**
- 桌面：3-4列
- 平板：2-3列
- 移动：1-2列

**对齐方式**
- 主要内容：左对齐或居中
- 导航：Logo左对齐，菜单右对齐
- Hero区：居中对齐

## 10. 视觉禁忌

### 10.1 避免使用

**颜色**
- 过于鲜艳的颜色
- 过多渐变色
- 低对比度配色

**布局**
- 过于拥挤的内容
- 不一致的间距
- 混乱的层级

**动画**
- 过于夸张的动画
- 过长的过渡时间
- 分散注意力的效果

### 10.2 推荐做法

**颜色**
- 使用主色调统一品牌
- 语义色明确用途
- 保持对比度充足

**布局**
- 充足留白
- 清晰层级
- 一致间距

**动画**
- 微妙精致
- 时间合理
- 增强体验
